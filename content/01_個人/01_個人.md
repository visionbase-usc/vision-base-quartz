# 01_個人

每位成員一個專屬資料夾，存放個人筆記、草稿、想法。

## 結構

```
01_個人/
├── <你的 GitHub username>/
│   ├── README.md           自我介紹（建議）
│   ├── 想法/
│   ├── 草稿/
│   └── ...
└── <其他成員>/
```

## 編輯規則

- ✅ 自己的資料夾隨便編
- ❌ **不要編輯別人的個人資料夾**（即使是筆誤也先 ping 對方）

## 加入新成員

1. 在 `01_個人/` 建立資料夾，命名用你的 GitHub username
2. 放一個 `README.md` 簡單介紹自己（姓名、研究主題、聯絡方式）
3. commit + push

## 隱私

vault repo 是私人的（只有 `visionbase-usc` org 成員看得到），但**別放真正敏感的東西**（密碼、token、未公開研究結果）—— Git history 留得很久。
