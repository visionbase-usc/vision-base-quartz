# 04_知識庫

實驗室共用的知識資源。**文獻摘要 + 技術筆記都放這**（不分開）。

## 結構

```
04_知識庫/
├── digital-twin/           Digital Twin 相關
├── physical-ai/            Physical AI 相關
├── 3d-reconstruction/      3D 重建、Point Cloud 等
├── tools/                  工具、平台、軟體相關筆記
└── (主軸延伸時再加)
```

## 子資料夾命名

英文 kebab-case，對應研究領域。新增子資料夾前在群組討論。

## 這裡放什麼

| 類型 | 例子 |
|---|---|
| 論文摘要 | `<paper-slug>.md` — 一篇論文一個檔，三句話 + 我的看法 |
| 技術 how-to | `mesh-reconstruction-pipeline.md` |
| 工具筆記 | `tools/blender-python.md` |
| 概念整理 | `gaussian-splatting-overview.md` |
| 外部資源整理 | `learning-resources-physical-ai.md` |

## 這裡**不放**什麼

- ❌ PDF 本體（>5MB） → 雲端，本檔貼連結
- ❌ 個人想法、未驗證的草稿 → `01_個人/<你>/`（成熟後再搬過來）
- ❌ 特定專案的內部討論 → `02_專案/<專案>/`

## Tag 使用

每篇建議標 `area/<領域>` 對應放置的子資料夾。例如放在 `digital-twin/` 的就標 `#area/digital-twin`，方便跨資料夾搜尋。

## 編輯規則

- 自由貢獻新檔案
- 改別人寫的內容前 ping 一下（小修錯字除外）
