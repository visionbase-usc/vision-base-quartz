# vision_base_vault 使用規範

VISION BASE Lab 多人共筆 vault 的使用規範。研究主軸：**Digital Twin + Physical AI**。

> 本文為人類成員主要參考；給 Claude Code 看的精簡版在根目錄 [`CLAUDE.md`](../CLAUDE.md)。

---

## 🗂 資料夾結構

```
vision_base_vault/
├── 00_共用/              全員共用文件、本規範、索引
├── 01_個人/<你的名字>/   個人筆記、草稿
├── 02_專案/<專案名>/     各專案資料
├── 03_會議/              跨專案、實驗室層級會議
├── 04_知識庫/            文獻 + 技術筆記
│   ├── digital-twin/
│   ├── physical-ai/
│   ├── 3d-reconstruction/
│   └── tools/
├── 05_GitHub_Repos/     visionbase-usc 各 repo 的文件鏡像
├── 06_CLAUDE_Skills/    Claude Code skills 的文件鏡像
├── _templates/           模板（會議 / 專案 / 筆記）
└── _attachments/         圖片、附件（<5MB）
```

### 各區編輯規則

| 資料夾 | 誰可以編輯 |
|---|---|
| `00_共用/` | 全員可，**改前在群組講一聲** |
| `01_個人/<你的名字>/` | 只有你自己，**別碰別人的個人區** |
| `02_專案/<專案>/` | 專案負責人主寫，其他人 PR / 提議 |
| `03_會議/` | 該場會議的紀錄者 |
| `04_知識庫/` | 自由貢獻，但**改別人寫的內容前** ping 一下 |
| `05_GitHub_Repos/` | 任何人可新增 / 更新自己負責的 repo 文件 |
| `06_CLAUDE_Skills/` | 管理員（程式檔在 `claude-skills` repo）|
| `_templates/` | 管理員 |

## 📋 YAML Front-matter

新筆記模板會**預設帶**這 4 欄。不想填可以刪，不會擋 commit。

```yaml
---
title: 筆記標題
author: 你的 GitHub username
created: 2026-05-24
tags: [type/note, status/draft]
---
```

為什麼有這個：
- 之後用 Dataview 外掛做索引、搜尋方便
- Claude Code 寫筆記時會自動填，多人共筆格式一致

## 🤝 共筆守則

### 衝突預防

**唯一策略：群組默契** —— 要改 `00_共用/`、`03_會議/`、`04_知識庫/` 裡的共用檔案前，在 Slack/Discord 講一聲。

個人區、自己負責的專案區隨便改。

### 如果還是發生 merge conflict

1. 群組喊一聲：「`<檔名>` 衝突，找 `<相關人>`」
2. 打開 Obsidian → `Ctrl+P` → `Obsidian Git: Open source control view`
3. 找到衝突檔案，編輯器會看到 `<<<<<<<`、`=======`、`>>>>>>>` 標記
4. 手動選要留的版本，刪掉衝突標記
5. `Obsidian Git: Commit all changes`

### 編輯節奏

- Obsidian Git 預設每 **5 分鐘**自動 commit + push + pull
- 長時間離線（一天以上）後，**先打開 Obsidian 讓它 pull 完**再開始編輯
- 編輯共用檔時最好先 `Ctrl+P` → `Obsidian Git: Pull` 一次確保最新

## 📦 哪些東西**不放** vault

| 類型 | 為什麼 | 放哪 |
|---|---|---|
| ❌ 程式碼、Jupyter notebook、訓練腳本 | git diff 難看，會把 vault 變沉 | 另開 code repo，vault 只放連結 |
| ❌ 訓練資料、感測器 log、量測數據 | 檔案大、頻繁變動 | 外部 NAS / 雲端，vault 寫結果 |
| ❌ 大檔案 >5MB（3D 模型、Point Cloud、CAD、IFC、影片）| GitHub 50MB 上限、會拖垮 clone 速度 | 外部 Drive/NAS，vault 放連結 + 縮圖 |
| ❌ 密碼、token、API key | 安全 | 密碼管理工具（1Password 等）|
| ❌ 學生個資、未授權研究原始資料 | 法規 + 倫理 | 不放任何 git repo |

論文 PDF 怎麼放：**之後再討論**。目前可以暫時放雲端，vault 寫摘要 + 連結。

## 🤖 Claude Code 整合

實驗室成員會大量使用 Claude Code 寫筆記、整理文件。為了讓 AI 寫的內容跟人寫的格式一致：

- **vault 根目錄有 [`CLAUDE.md`](../CLAUDE.md)**，Claude Code 開啟 vault 時會自動讀
- 裡面寫了：folder 結構、tag 分類、front-matter 範本、命名規則、不放 vault 的東西
- Claude Code 寫筆記會自動套用這些規則

### 配套 Skill

[`visionbase-usc/claude-skills`](https://github.com/visionbase-usc/claude-skills) repo 內未來會有專為本 vault 設計的 `vision-base-vault` skill。安裝後 Claude Code 處理 vault 相關任務更一致。

各 skill 的文件鏡像在本 vault 的 [`06_CLAUDE_Skills/`](../06_CLAUDE_Skills/) 資料夾，方便離線查閱。

org 內所有 repo 的文件鏡像則在 [`05_GitHub_Repos/`](../05_GitHub_Repos/)。

## 📥 新成員 Onboarding

1. **GitHub 帳號加入 `visionbase-usc` org**（管理員邀請）
2. 照 vault 根目錄 [`README.md`](../README.md) 第 1 段裝工具（Obsidian / git / gh CLI）
3. `gh auth login` 登入個人 GitHub 帳號
4. `git clone https://github.com/visionbase-usc/vision_base_vault.git`
5. Obsidian → Open folder as vault → 關掉「限制模式」→ 啟用 Obsidian Git
6. 設定 Obsidian Git 自動同步間隔（5 分鐘）
7. 在 `01_個人/` 建立自己的資料夾，放一個 `README.md` 寫自我介紹
8. 讀完本規範 → 跟管理員回報
