## 一、整體結構概況

硬碟採「**編號分類法**」當骨架，這是好的設計：

| 編號 | 資料夾 | 狀態 |
|---|---|---|
| 00 | 行政文件 | ✅ |
| 01 | Proposal | ✅ |
| 02 | 設備 & 軟體報價 | ⚠️ 2023 後幾乎沒動 |
| 03 | 空間建置 | ⚠️ 2023 後沒動 |
| 04 | Research | ✅ |
| 05 | 專案 | ✅ 主力 |
| 06 | 活動講座 | ✅ |
| 07 | Dev Team | ✅ |
| 98 | Dev（含 Omniverse） | 🆕 6/17 新建 |
| 99 | 知識庫 | ✅ |

**核心問題：這套編號制度只蓋住一半。** 根目錄同時還散落一堆「沒有編號」的資料夾與檔案（如 `202608 地下美術館展覽`、`一般型計劃`、`一審後修改`、`2025大肆設計課`、`***VISIONBASE INTRODUCTION***`），分類規則沒有被貫徹。

## 二、四大結構問題（依嚴重度排序）

### 🔴 1. 根目錄變成「垃圾暫存區」

根目錄直接躺著大量本該歸檔的散檔，這是目前最嚴重的結構問題：

- **散落照片 ~28 張**：`IMG_0189`～`IMG_0214`（一整批，約 95MB）、`IMG_0944`、`IMG_0956`
- **散落影片 3 支**：`1.mp4`（292MB）、`生活2.mp4`、`2.mp4` — 共約 335MB 裸放在根目錄
- **散落單檔**：`555.3dm`、`寄件封面.docx`、各種收據 doc、PDF

👉 根目錄應該只放資料夾 +（最多）少數索引檔。

### 🔴 2. 「未命名」檔案氾濫（約 17 個）

根目錄至少有 **17 個** `未命名文件 / 未命名簡報 / 未命名的試算表`，其中含真實內容的大檔：

- `未命名文件`（31MB，2025-11）
- `未命名文件`（10MB，2025-12）
- `未命名文件`（597KB）

👉 這些幾乎不可能用搜尋找到，等同「資料黑洞」。

### 🟠 3. 副本 / 版本混亂

- `以下檔案的副本： 盧組`（**121MB** 簡報，純副本沒改名）
- `徐婷＿細部設計計畫書20260515.pptx 的副本`（30MB）
- `用印申請表 的副本`
- 共用區更嚴重：`【流形】…的副本 的副本`、`…的副本 的副本 的副本`（三層副本）

👉 沒有版本命名規則（v1 / v2 / final），靠「的副本」堆疊。

### 🟠 4. 命名規則不一致

- 中英混雜、有無日期不一（`VISIONBASE工作內容報告20260601` ✅ vs `說明文字稿` ❌）
- **錯字**：`專任肋理`（應為「助理」）、`一般型計劃`（應為「計畫」）
- 特殊符號裝飾：`***VISIONBASE INTRODUCTION***`（影響排序與搜尋）

## 三、🔐 安全 / 權限提醒（重要）

> ⚠️ 具體檔名與位置已自本（公開同步）文件移除，請見內部頻道。

- 硬碟中存在**含敏感登入資訊**的文件。
- 行動 1：確認該類文件的共用權限**非「知道連結的人皆可檢視」**。
- 行動 2：理想上改用密碼管理工具（如 1Password / Bitwarden），勿以雲端純文字檔保存帳密。
