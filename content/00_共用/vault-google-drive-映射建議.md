# vault ↔ Google Drive 映射建議

> 本文定義 Obsidian **vault** 與團隊 **Google Drive** 兩套儲存系統的分工與對應關係，避免兩邊重複存檔、互相找不到。為待討論的規範草案。

---

## 一、核心原則：各管一邊、互相連結、不重複存

| 系統 | 是什麼的「單一真實來源」(SSOT) | 特性 |
|---|---|---|
| **vault** | **文字、知識、決策、規範** | 純 markdown、版本控管 (Git)、可全文搜尋、會同步到公開 Quartz 站 |
| **Google Drive** | **重檔、正本、行政財務** | 二進位大檔（影片 / 簡報 / 圖庫 / CAD / zip）、官方簽核版本、權限分享 |

**三條鐵則：**
1. 一份內容只在一邊當正本，另一邊用**連結**指過去。
2. 文字寫在 vault；檔案（>5MB 或需簽核 / 分享）放 Drive。
3. vault 筆記頂部釘住對應的 Drive 連結；Drive 資料夾放一個指回 vault 的說明。

---

## 二、資料夾對應表

> ⚠️ 注意：兩邊都用 00–06 編號，但**意義不同**（見第三節對照），別把編號當對齊依據。

| vault 資料夾 | 角色（文字 SSOT） | ↔ | Google Drive 資料夾 | 角色（檔案 SSOT） |
|---|---|---|---|---|
| `00_共用/` | 規範、索引、治理文件 | ↔ | `00_行政文件/` | 行政正本、簽核 PDF |
| `01_個人/<name>/` | 個人筆記、草稿 | ↔ | （個人雲端，不強制） | — |
| `02_專案/<project>/` | 專案知識、筆記、決策 | ↔ | `05_專案/<同名>/`、各散落專案夾 | 專案交付物、簡報、素材 |
| `03_會議/` | 會議記錄（文字） | ↔ | （Drive 對應夾） | 會議錄影 / 錄音 / 簡報原檔 |
| `04_知識庫/` | 文獻摘要、技術筆記 | ↔ | `04_Research/`、`99_知識庫/` | 論文 PDF 正本、原始資料 |
| `05_GitHub_Repos/` | repo 文件鏡像 | ↔ | （無，GitHub 為正本） | — |
| `06_CLAUDE_Skills/` | skills 文件鏡像 | ↔ | （無） | — |
| `_attachments/`（<5MB） | 筆記用小圖 | ↔ | Drive 圖庫（>5MB） | 大圖、影片、原檔 |

---

## 三、編號對照（避免混淆）

vault 與 Drive 的同號資料夾**不是同一件事**，引用時請註明來源：

| 概念 | vault | Drive |
|---|---|---|
| 專案 | `02_專案` | `05_專案` |
| 知識庫 | `04_知識庫` | `99_知識庫` + `04_Research` |
| 行政 / 共用 | `00_共用` | `00_行政文件` |
| 會議 | `03_會議` | （散落於各專案） |

**引用寫法**：用前綴標明系統，例如 `vault:02_專案/光沐` 與 `drive:05_專案/光沐`，避免口頭講「02 那個」造成誤會。

---

## 四、跨系統連結慣例

### vault 筆記 → Drive 檔案
在筆記頂部用統一格式釘連結（建議加 📁 前綴方便辨識）：

```markdown
> 📁 **Drive 資料夾**：[光沐專案 (Drive)](https://drive.google.com/drive/folders/XXXX)
> 📎 相關正本：[提案簡報 v3 (Drive)](https://docs.google.com/presentation/d/XXXX)
```

### Drive 資料夾 → vault
在 Drive 專案資料夾內放一個 `_vault.txt` 或 Google Doc，內容只有一行：

```
本專案的筆記 / 決策記錄在 vault：02_專案/<專案名>/
```

## 五、決策規則：「這東西該放哪？」

| 內容 | 放哪 |
|---|---|
| 筆記、決策、規範、討論結論 | 🟢 **vault** |
| 需要簽核 / 對外的正式文件 | 🔵 **Drive**（`00_行政文件` / `01_Proposal`） |
| 論文 PDF 本體 | 🔵 **Drive**，摘要寫進 🟢 vault `04_知識庫/papers/` |
| 圖片 < 5MB 且筆記要用 | 🟢 **vault** `_attachments/` |
| 圖片 / 影片 / 簡報 / CAD / zip（>5MB） | 🔵 **Drive** |
| 帳密、含敏感登入資訊的文件 | 🔴 **都不放純文字** → 密碼管理工具 |

> 📌 vault 會同步到**公開** Quartz 站，**敏感 / 內部營運細節不要寫進同步資料夾**，或標 `draft: true`（僅擋渲染，原始檔仍進公開 repo），真正敏感的請放同步清單外的資料夾。
