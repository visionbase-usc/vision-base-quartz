# 隨行控制平板 · 展務員跨區控制台

**定位**:全程在現場導覽員手上,**訪客看不到**。用途不是功能展示,是**穩定整體展覽流程**——在卡關時介入幫忙推進(卡片故障、訪客不會操作、畫面卡住、設備掉線)。

由這個定位推出按鈕優先序:
1. **異常排除 / 推進流程**(最重要)— 重置、強制推進、強制換頁
2. **講解備案** — 不靠實體卡手動觸發 demo
3. **設備監看** — 連線總覽,掛了第一時間知道

原則:訪客能自助的不放這裡。每做完一區,在下面那一區補上:需要哪些按鈕 + server 在哪 + 後端 hook 做了沒。

---

## 各區按鈕

### F — AI 大腦控制塔 ✅
- **server**:`ws://localhost:8787`(F 區桌面/牆面/桌上平板共用同一個)
- **repo**:`VistwinProject/F-table`
- **訪客自助(不歸這支)**:開始體驗 = 訪客點桌上固定平板;刷卡 = NFC 自動觸發
- **這支要的按鈕**(依優先序):

| 優先 | 按鈕 | 送出訊息 | 後端 hook |
|---|---|---|---|
| 1 異常 | 重置 F 區 | `{type:"session-end"}` → 三畫面清空回歡迎頁(訪客留卡/卡住時一鍵重來) | ✅ 已做 |
| 1 異常 | 強制開始 | `{type:"session-start"}` → 三畫面進互動頁(訪客不會點桌上平板時代按) | ✅ 已做 |
| 2 講解 | 手動觸發家電 | `{type:"manual-trigger", id:"<id>"}` → 三畫面當作被刷卡亮起 | ⏳ 未做(`server.py`) |
| 2 講解 | 清除手動 | `{type:"manual-clear", id:"<id>"}` 或全清 → 等同 tag-remove | ⏳ 未做 |

- session-start / session-end 已實作:任何 client 送都會廣播,server 記住狀態(見 [[F-AI大腦控制塔]] §三畫面同步規格)。導覽員代按 = 跟訪客點桌上平板同一條路徑。
- `id` = 7 個 freeze 字串:`door / ac / light / socket / curtain / sound / hrv`
- manual hook 規格:合成事件佔虛擬 slot(`slot_index=-1`),不撞實體 reader。

## 跨區共用按鈕(不歸單一區)

| 按鈕 / 顯示 | 作用 |
|---|---|
| 區域切換 tab | 選目前控哪一區 |
| 全區連線總覽(顯示) | 各區 桌面/牆面/平板/server 在線燈號 |
| 重置單區 | 清掉某區當前互動狀態(訪客亂留卡片、卡住) |
| 緊急:全部回首頁 | 所有區回歡迎/待機 |
| 緊急:全部停止 | 全場停(慎用) |
