# 260608 交付速覽

> 來源：260608 錄音整理。重點 = 各系統（A / B / F / TD）的 **Demo 規劃、自動化執行、長期運行維護**。

## TL;DR

- **A 區**：轉場（專場）可不順，但**必須能跑完整動作序列（sequence）** — 開機 → 感應到人 → 各階段運行秒數 → 影片銜接（影片可用暫時素材替代）。
- **B / F 區**：難度低，採**最簡解**（`fadein / fadeout`）即可。
- **部署目標**：像 iPhone / ATM 一樣 **24h 常開**，開機自動載入對應螢幕畫面。
- **穩定性**：連跑 30 天恐爆記憶體 → **每天重啟一次**。
- **操作**：現場人員只要**開一個檔（整個殼）就能啟動**。

## 1. Demo 展示要求

### A 區 → [[A-數據之門]]
- 轉場不順可接受，**重點是完整 sequence 能展示**：
  - 系統開啟
  - 感應到人來的互動
  - 各階段運行秒數
  - 影片銜接（內容可先用暫時素材）

### B 區 → [[B-感應光寓]]、F 區 → [[F-AI大腦控制塔]]
- 認定**較無難度**，用最簡單解法處理。
- 轉場 / 銜接走 **fadein → fadeout**。

## 2. 技術執行與部署

- **自動化啟動**：24h 常開，開機後自動載入對應螢幕畫面。
- **批次處理（BAT）**：B / F 的交接與啟動寫成 **`.bat` 檔**，附 **README** 說明環境安裝步驟。
- **多螢幕 / TD 設定**：
  - 目前用瀏覽器執行，**多螢幕的辨識與對應**較麻煩 → **需現場測試**。
  - TouchDesigner（TD）自動執行 → 後續再研究如何簡化啟動流程。

## 3. 穩定性與維護

- **記憶體管理**：24h × 30 天可能爆記憶體 → **建議每天重啟一次**。
- **操作簡便化**：啟動極簡，最好**開一個檔（整個殼）就完成**，讓現場工作人員好處理。
