# 期中現況盤點與 VISION BASE 建議

> 計畫：**AI 加值服裝設計數位流程建構與應用實證計畫**（國科會 114WFDA210072）
> 用途：期中審查內部討論 ＋ 給設計學院院長的 VISION BASE 建議
> 簡報檔：[[refs/2026-06-15_期中現況與建議.pptx]]

## 計畫定位

用生成式 AI 把服裝設計流程「草圖 → 靜態擬真圖 → 動態提案影片」串成一條可複製、可教學的數位流程，與 **聚陽實業（製造端）** ＋ **UUIN／赤赤子（品牌端）** 做場域驗證，並導入大二課程培育人才。

- 主持人 = 許鳳玉（教授兼設計學院院長）；共同主持人 = 盧彥臣（VISION BASE 專案經理）
- 補助 450 萬／一年期（114/12 – 115/11）
- **VISION BASE 是本計畫的 AIGC 技術引擎**——委員質疑「技術創新性」時要負責回答的就是這一塊

## 二、委員五大可能質疑（推測，非原文）

1. **經費執行率僅 13%** — 進度落後、能否如期完成？（最可能被問）
2. **效益數字未經場域實證** — 打樣 −15%、開發 −22% 等數字，A/B 驗證還沒跑（實務型計畫核心）
3. **技術創新性不足** — 整合 14+ 種現成 SaaS，跟一般人用 AI 工具差在哪？（延續初審意見 1、14）
4. **平台化名實不符** — 一直講「中台／系統」，但平台在哪？
5. **智財／版權／資安** — 承諾的「地端自訓 LORA」未見，免費 SaaS 有機敏資料外流風險

## 三、平台化問題（落差最大的一塊）

| 計畫書承諾 | 期中實際交付 |
|---|---|
| AI 視覺決策「中台」、串接協作平台、系統性整合、消滅「資料孤島」 | 一張流程圖 + 技術規格書、SOP 手冊 + Prompt CSV、14+ 種 SaaS 靠**人工**串接 |

簡報困難一自承：跨工具串接需「額外人工調整」＝**平台（自動整合）這件事還沒做出來**。沒有平台，「可複製、可導入其他品牌」的擴散性說法就軟了。

## 四、VISION BASE 三大建議（一石三鳥）

用 **ComfyUI 地端整合層** 當平台 MVP，同一個技術動作回應委員三個質疑：

- **解決平台化** — ComfyUI 節點式 workflow 把「文生圖→材質→套版→影片」串成可存檔、可重跑的 pipeline，取代人工串接
- **解決技術創新** — 用聚陽私有資料在 RTX 4080 自訓 LORA、跑地端 SDXL/Flux，不是只排列 SaaS 工具
- **解決版權/資安** — 封閉地端部署，企業機敏資料不外流，補上對委員承諾過的合規機制

> **定調**：不做 SaaS 產品；做「地端、可自訓、把零散工具標準化串接的整合層」——這是 VISION BASE 的技術角色。

## 五、下半年優先順序

1. **Q3（最優先）** — 聚陽 + UUIN 場域真的跑起來，取真實 A/B 對照數據
2. **Q3** — ComfyUI 地端 pipeline + LORA 自訓 demo
3. **Q3–Q4** — 案例轉教案、導入大二課程
4. **Q4** — 種子教師工作坊、成果發表會

**守審查三動作**：① 主動解釋 13%（前期建構、支出集中下半年）② 效益數字標為「場域實作前預估」③ 平台用詞收斂。
