# 02_專案

實驗室各專案的文件區。每個專案一個資料夾。

## 結構

```
02_專案/
├── <project-slug>/
│   ├── README.md           專案概述（必有）
│   ├── 會議/                該專案的會議記錄
│   ├── 筆記/                技術筆記、進度
│   ├── refs/                該專案相關文獻、外部參考
│   └── _attachments/        該專案專用的圖片（小檔）
```

## 命名

專案資料夾用**英文 kebab-case**，例：
- `twin-shihchien-campus`
- `physical-ai-robotics`
- `point-cloud-pipeline`

中文專案名請寫在資料夾內的 README.md 標題。

## 專案 README.md 建議內容

可參考 [`_templates/專案/README.md`](../_templates/專案/default.md)，包含：
- 專案名稱（中英文）
- 負責人 / 成員
- 目標、成果交付
- 時程
- 相關連結（外部 code repo、論文、雲端資料夾）

## 編輯規則

- 專案**負責人**主寫
- 其他成員想改：先在群組講一聲，或在 vault repo 開 issue 提建議
- 大改動建議走「先在個人區草稿 → 確認後合進專案區」的方式

## 程式碼怎麼辦

**不放 vault**。每個專案另開 GitHub repo，在專案 README.md 裡放連結。理由：
- Notebook / code 用 git diff 看很痛苦
- 程式碼變動頻繁，會把 vault 搞得很重
- vault 的價值在於「人讀」的內容
